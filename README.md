# 솜씨 (Somssi) — MVP (PRD v3.1 / UX flow v2)

> 해결하고 싶은 문제·아이디어가 있는 **비전문가**가, 개발·디자인을 잘 만드는 사람과
> 짧게 한 팀을 이뤄 첫 작동 버전을 내놓도록 돕는 서비스.
> AI가 ① 필요한 역할을 짚고 ② 실력 신호로 사람을 골라준다.

UI/UX는 **솜씨 UX flow v2**를 기준으로 구현했습니다 — Pretendard 폰트, 흙톤 캔버스 배경에
**그린(주색)·테라코타·퍼플** 퍼스널 컬러(색채심리), 라인 SVG 아이콘, 바텀시트, 탭바.
모바일 우선(데스크톱은 폰 프레임).

## IA (정보구조) 매핑

업로드된 IA 표(구분 0~6)를 그대로 화면 구조로 구현했습니다.

| # | 구분 | 화면 | 접근권한 |
| --- | --- | --- | --- |
| 0 | Landing | 슬로건·여정 소개·현황 지표·회원가입 CTA | 비로그인 |
| 1 | 서비스 진입(분기) | 로그인 게이트 → 팀원 구하기 / 되기 | 로그인 필요 |
| 2 | 수요 플로우 | 아이디어 → AI 가이드(타겟·핵심기능·규모) → 브리프(확인·수정) → 부합도 매칭 → 같이하기→수락→상세공개 → 성사 | 수요자 |
| 3 | 공급 플로우 | 역할 선택(3개) → 실력 신호 등록(결과물·인터뷰·평판) → 받은 같이하기 요청(수락/거절) | 공급자 |
| 4 | My Dashboard | 활성도·신뢰 신호 / 내 프로젝트·매칭 현황 / 진행 패널(브리프→…→출시, NSM·KPI) | 로그인 공통 |
| 5 | My Information | 프로필, 알림센터(요청·수락), 계정 설정(보류) | 로그인 공통 |
| 6 | 운영자(Admin) | 전문가 검수·실력 인증 부여, KPI·퍼널 측정, 기관 제휴(보류) | 운영자 |

**범례/결정사항 반영:**
- **역할 후보 = 정확히 3개**: 서비스 기획자 / UX·UI 디자이너 / 노코드·풀스택 개발자 → AI 매칭 `role` enum 고정(`/api/roles`), 공급 등록도 3개 중 선택
- **접근권한**: 소개=비로그인 / 서비스 진입부터 로그인 필요(게이트 바텀시트)
- **온보딩**: 한 줄 입력 → 객관식 바텀시트(같은 페이지) → 브리프 자동완성·확인·수정(1분)
- **프로필**: 선택 전=간단(이름·역할·인증·부합%) / 수락 후=상세(결과물·인터뷰·평판, 양쪽 교환)
- **보류(이번 Out of Scope)**: 이용 사례, 아이디·비번 찾기, 약관, 계정 설정, 기관 제휴 → "준비 중" 표시
- **가설 매핑**: H1(역할 짚기·브리프), H2(추천→선택), H4(인증·공급), H5(최소 입력)
- **퍼스널 컬러**: 그린=기획/시스템, 퍼플=디자인/AI, 테라코타=개발/연결

| PRD 5.2 기능 | 구현 위치 |
| --- | --- |
| 전문가 DB (결과물·인터뷰·평판) | `data/makers.json`(+`role`) + `/api/makers` |
| AI 역할 짚기 + 매칭 (H1·H2·H5) | `/api/recommend` (brief + role-enum 구조화 출력) |
| 대시보드 / 측정 | `/api/stats`, `/api/projects` |
| 실력 인증 (✓) | `verified` 필드, 운영자 `/api/makers/:id/verify` |

**UX flow v2 화면 흐름:**

- **탭바**: 소개(비로그인 OK) / 서비스(로그인 필요 → 로그인 게이트 바텀시트)
- **분기**: 팀원 구하기(그린) / 팀원 되기(테라코타)
- **수요 여정**: 아이디어 한 줄 → AI 가이드 질문(타겟) → **매칭 브리프**(문제·타겟·필요역할)
  → **부합도 매칭**(역할별 후보 + 부합% + ✓인증) → **같이하기 → 수락 → 상세 프로필 공개**
  → **팀 결성**("솜씨는 여기까지 — 만들기는 팀이")
- **공급 여정**: 전문가 등록(최소 입력) → 운영자 검증 후 ✓인증

**색채심리(퍼스널 컬러):** 그린=신뢰/완료, 테라코타=만드는 사람/연결, 퍼플=AI 가이드.

**Out of Scope(3) 준수:** 무엇을 만들지·작업 범위는 정하지 않음. 추천만 한다.
팀 결성 이후(단기 협업)는 플랫폼 밖 여정이며, 성공 기준에서 측정만 한다(`teamFormed`).

## 실행

```bash
npm install        # 최초 1회
npm start          # 서버 실행
```
→ 브라우저 http://localhost:3000 (모바일 화면 비율로 보여요)

## 화면 흐름

- **홈**: 수요/공급 분기 + 실시간 대시보드(프로젝트·전문가·인증·결성 수)
- **수요(팀원 구하기)**: 아이디어 한 줄 → AI가 역할 2~3개 + 역할별 추천(점수·이유·✓인증) → 팀원 선택 → 팀 결성 완료(Aha)
- **공급(팀원 되기)**: 최소 입력으로 전문가 등록 → 운영자 검증 후 인증 마크

## API

| 메서드 | 경로 | 설명 |
| --- | --- | --- |
| GET | `/api/roles` | 역할 enum(3개) |
| GET | `/api/stats` | 대시보드/KPI 현황 |
| GET | `/api/projects` | 최근 프로젝트(내 현황) |
| GET | `/api/makers` | 전문가 DB |
| POST | `/api/makers` | 전문가 등록(공급, 역할 포함) |
| POST | `/api/makers/:id/verify` | 운영자 실력 인증 부여/해제 |
| POST | `/api/recommend` | 아이디어+가이드 → 브리프 + 역할 짚기 + 부합도 매칭 |
| POST | `/api/team` | 팀 결성(성사) 기록 |

## 기술

- 백엔드: Node.js + Express, 파일 기반 DB(`data/*.json`)
- AI: Claude API (`claude-opus-4-8`), 구조화 출력(`output_config.format`)
- 프론트: 바닐라 HTML/CSS/JS, 토스 스타일 단계 플로우
- 키: `.env` 의 `ANTHROPIC_API_KEY`

## 파일 구조

```
server.js              # API 서버
data/makers.json       # 전문가 DB (verified=인증)
data/projects.json     # 프로젝트/팀결성 기록(대시보드·NSM)
public/
  index.html           # 폰 프레임 셸
  style.css            # 토스 디자인 토큰
  app.js               # 단계 상태머신(분기→입력→매칭→결성, 공급 등록)
.env                   # ANTHROPIC_API_KEY (커밋 금지)
```

## 성공 기준 측정(7)

- `teamFormedCount / projectCount` → 매칭→결성 전환
- 팀 결성까지 클릭 수 = 3회(설계상 고정)
- 단기 협업 착수율·출시율(Aha)·CSAT 는 플랫폼 밖 측정 대상
